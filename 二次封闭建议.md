# 二次封口优化建议

## 背景

二次封口（`execute_seal`）在实心化后对 Skin 段执行两次 MORPHOLOGICAL_CLOSING，目的是封闭耳道和鼻孔等小开口，防止 EDT 沿腔道内壁生长出错误的 bolus。

## 当前问题

```
实心化 → 显示 seal 滑块（默认 10/5）→ 执行 → 切到 Slicer 看效果 → 不满意？再次执行（kernel 叠加肿胀）
```

| # | 痛点 | 影响 |
|---|---|---|
| 1 | 看不到代价 | 滑 kernel 时不知道实际会压平多少 |
| 2 | 没场景预设 | 每次手动判断 8/4 还是 12/6 |
| 3 | 没客观反馈 | 执行后只能肉眼看，没有数据说明封没封住 |
| 4 | 多次执行累积 | 第二次 closing 基于已 sealed 的 skin，越来越肿 |
| 5 | 没对比机制 | 无法直观看 before/after 差异 |

## 精度影响量化

`MORPHOLOGICAL_CLOSING kernel = K mm` 的理论最大位移：

```
最大局部变形 ≈ K / 2
```

| 场景 | 推荐 kernel | 耳廓最大压平 | 鼻翼最大失真 |
|---|---|---|---|
| 头顶/前额 bolus（耳鼻在 ROI 外） | **10 / 5**（默认） | ~5mm（无所谓） | ~2.5mm（无所谓） |
| 全脸 bolus 精度优先 | **6 / 3** | ~3mm | ~1.5mm |
| 全脸 bolus 封口优先 | **15 / 8** | ~7.5mm | ~4mm |
| 颈 / 躯干 / 四肢 | **跳过** | 0 | 0 |

> 注：CLOSING 保凸面，压平凹陷。耳廓"压平"指耳轮/对耳轮沟被填，外形变椭圆。

## 改进方案（A–F，按优先级）

### A. 实时精度提示（5 行前端，无后端改动）

Slider 旁动态显示当前 kernel 对应的最大失真量：

```jsx
<p className="text-xs text-medical-500">
  当前大核 {k1}mm → 耳廓最大压平 ~{(k1/2).toFixed(0)}mm
</p>
<p className="text-xs text-medical-500">
  当前中核 {k2}mm → 鼻翼最大失真 ~{(k2/2).toFixed(0)}mm
</p>
```

**收益**：用户滑动时立即看到精度代价，无需心算。

### B. 场景预设按钮（15 行前端，无后端改动）

```
[头顶/前额]  10/5    推荐默认（耳鼻在 ROI 外，精度损失无所谓）
[全脸保守]   6/3     耳鼻精度优先（小开口可能漏封，用户自查）
[全脸标准]   12/6    耳道确保封闭（耳廓精度牺牲较大）
[跳过]        —      颈/躯干/四肢/后背（无开口，直接跳过）
```

点按钮一键应用 `seal_kernel_1_mm` / `seal_kernel_2_mm`，slider 随之更新。

**实现位置**：`SegmentationPanel.tsx` solidified 状态的封口参数区块上方。

### C. 客观反馈（30 行后端 + 前端显示）

执行 `execute_seal` 后，后端额外返回：

```python
# execute_seal 返回值新增：
{
  "residual_cavities_cm3": 0.2,   # 残余内部气腔（=0 最好）
  "max_surface_shift_mm": 2.8,    # 皮肤表面最大位移
  "mean_surface_shift_mm": 1.1,   # 皮肤表面平均位移
}
```

**后端实现**：
1. Closing 前记录 Skin labelmap → 存为 numpy
2. Closing 后重新读取 → 比较差异
3. 用 connected_components 算残余内部气腔体积
4. 用 distance_transform_edt 算位移场均值/最大值

**前端展示**（sealed 状态）：
```
✓ 气腔已完全封闭（0.0 cm³残余）  | 表面最大位移 2.8mm 均值 1.1mm
⚠ 残余气腔 0.2 cm³（建议大核 +2mm 后重试）
```

**收益**：不用切 Slicer 就知道结果，有具体数字指导下一步。

### D. 多次执行不累积（20 行后端）

**问题**：目前 `execute_seal` 每次在当前 skin 上叠加 closing，多次执行肿胀累积。

**方案**：
- `execute_solidify` 完成后，将 Skin 备份到 `Skin_Solid_Backup` segment（不显示）
- 每次 `execute_seal` 开始时先用 `Skin_Solid_Backup` 还原 Skin，再执行 closing
- 用户无论调多少次 kernel，都基于实心化后的干净状态迭代

```python
# execute_solidify 末尾：
backup_id = seg.AddEmptySegment("Skin_Solid_Backup", ...)
editorWidget.apply(Logical operators, COPY, src=skin_id → backup_id)
seg.GetSegment(backup_id).SetIsHidden(True)

# execute_seal 开头：
if backup_id := seg.GetSegmentIdBySegmentName("Skin_Solid_Backup"):
    apply(Logical operators, COPY, src=backup_id → skin_id)  # 先还原
# 然后正常执行 closing...
```

**收益**：kernel 从 10 改到 6 重试，结果不累积，UI 改参再按就够了。

## 推荐组合

### 最优（全体验）

**A + B + C + D**，约 70 行改动：

| 目的 | 方案 | 效果 |
|---|---|---|
| 看到代价 | A（5行） | 滑动时实时显示压平量 |
| 快速选场景 | B（15行） | 一键配置，无需手调 |
| 量化结果 | C（30行） | 执行后数字反馈 |
| 反复试不累积 | D（20行） | 换 kernel 自由迭代 |

### 最小有效

如果只做一项 → **D（撤销机制）**：最大改善工作流，解决"越封越肿"的根本问题。

如果只做两项 → **B + D**：场景预设 + 迭代不累积。
